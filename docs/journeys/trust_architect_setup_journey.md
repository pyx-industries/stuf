# Trust Architect Journey: Setting Up a New STUF Instance

## User
Trust Architect (TA)

## Goal
Create and configure a new Secure Trusted Upload Facility for a project

## Journey Steps

### 1. Initial Setup
- TA logs into the STUF Admin tool
- TA creates a new STUF instance with basic project details
- System provisions the necessary infrastructure
- TA receives confirmation when the instance is ready

### 2. User Management
- TA configures the list of authorized project participants
- TA assigns appropriate roles to each participant
- TA reviews the participant list for accuracy
- System sends invitation emails to participants

### 3. Metadata Configuration
- TA configures metadata requirements for uploads
- TA defines which metadata fields are required vs. optional
- TA sets up predefined options for metadata fields
- TA saves and tests the metadata configuration

### 4. Testing and Verification
- TA performs a test upload to verify functionality
- TA confirms notifications are working correctly
- TA verifies access controls are properly configured
- TA makes any necessary adjustments to the configuration

### 5. Monitoring and Management
- TA monitors initial uploads from participants
- TA accesses and reviews uploaded files
- TA manages any issues reported by participants
- TA makes configuration adjustments as needed

## Related User Stories
- TA_CREATE_STUF_001: Create new STUF instance
- TA_MANAGE_USERS_001: Configure authorized users
- TA_METADATA_001: Configure metadata requirements
- TA_FILE_ACCESS_001: Access uploaded files
- TA_MONITOR_001: Monitor upload activities

## Success Criteria
- STUF instance is properly configured and operational
- Authorized participants can successfully upload files
- Metadata is collected according to requirements
- TA can access and manage all uploaded files
- Notifications are delivered for all upload events
